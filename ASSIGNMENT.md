Remote API Challenge

Welcome! We want to see how you approach a realistic, bounded technical challenge.
The Task

Build a working organizational chart application that:

    Fetches data from Remote's public API (employee & company information)
    Displays the org chart clearly showing manager-employee relationships
    Shows key details like names, titles, departments, reporting lines
    Handles edge cases gracefully (missing data, employees with no manager, etc.)
    Gets deployed to a public URL so we can test it

Getting Started
Your Sandbox Credentials

You'll receive an email with:

    URL: https://employ.remote-sandbox.com
    Username: your-email@remote.com
    Password: (in the email)

Just log in and you're ready to go. The sandbox includes sample company data ("Acme Sandbox Corp") with realistic employees, titles, departments, and org structure.
API Documentation

    Remote API Docs: https://developer.remote.com
    Getting Started: https://developer.remote.com/docs/getting-started-with-remote
    Sample Code: https://github.com/remoteoss/remote-api-demo
    Remote MCP (Optional): https://developer.remote.com/docs/introduction-to-remote-mcp

What to Build
Requirements

Functional:

    Fetch employee and organizational data from Remote's API
    Display the org chart with hierarchical relationships (manager → direct reports)
    Show: name, title, department, and reporting line for each person
    Use one visualization format (tree diagram, interactive chart, table, etc. — your choice)
    Handle missing data and edge cases gracefully

Technical:

    Write clean, maintainable code with comments
    Include proper error handling
    Add a README explaining setup and how to run it
    Use any tech stack you're comfortable with (no restrictions)

What to Submit
Phase 1: Build (Your Code)

    Source Code: GitHub repo (or accessible code)
    Local App: Something runnable on your machine
    README with:
        How to set up and run the project
        Overview of your approach/architecture
        Any assumptions or limitations you made

Phase 2: Deploy (Public URL)

    Deploy your app to a public URL (Vercel, Heroku, AWS, etc. — your choice)
    Include deployment documentation
    Make sure we can access and test it

Phase 3: Share & Review

When ready, email the interview team:

    Link to your GitHub repo
    Link to your deployed app
    Brief summary of your tech choices
    Any special setup instructions

We'll review your code and application, then schedule a call to discuss it with you.
Tips

    Use AI Tools: We expect you to leverage AI (Claude, ChatGPT, etc.) to build this solution. Using AI tools effectively is a key skill we're evaluating. Show us how you work with modern tooling.
    Remote MCP: Consider using Remote's MCP (Model Context Protocol) for structured API access. It's available in the sandbox and can accelerate development.
    Scope: This is bounded — don't over-engineer. A working solution is better than incomplete perfection.
    Tech Stack: Use whatever you're most comfortable with. This isn't a test of specific frameworks.
    Get Unstuck: If you hit a blocker, reach out. We want to see how you handle problems, not whether you can figure out everything alone.
    Self-Paced: Work at your own pace and share results when you're ready.

FAQ

How long should this take?

For a solid developer, 4-6 hours of focused work. But work at your own pace.

Can I use a boilerplate or starter template?

Absolutely. Focus on the org chart, not scaffolding.

Do I need a database?

Only if you want one. The API data can be processed in memory for a demo.

What if the API is missing data I need?

Reach out — we can clarify what's available or help troubleshoot.

Can I deploy on my own infrastructure?

Yes, as long as it's publicly accessible.

What if I get stuck?

Email the interview team. We're here to help.

Should I use AI tools like Claude?

Yes, please. We expect you to use AI tools to build this solution. Show us how you work with modern tooling — that's part of what we're evaluating.

Can I use Remote's MCP?

Absolutely. It's available in the sandbox and can help you integrate with the API more efficiently. Using it shows you know how to leverage structured tool access.
